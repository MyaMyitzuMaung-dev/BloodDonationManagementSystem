Blood Donation Management System
1. Authentication & Authorization Module
Features: 
-Register User (Donor / Requester)
-Login
-JWT Token Generation
-Role-based Authorization (Admin / Donor / Requester)
-Logout
Roles:
-Admin
-Donor
-Requester

2. User Management Module (Admin)
Features:
-View all users
-Create / Edit / Delete user (soft delete)
-Activate / Deactivate user
-View user details
-Filter by Role (Donor / Requester)

3. Donor Management Module
Features:
-Register as Donor
-Update profile (blood group, location, availability)
-View donation history
-Check eligibility (LastDonationDate)
-Mark availability (Available / Not Available)

4. Blood Request Module (Core Feature ⭐)
Features:
-Create blood request (Requester)
-View all requests
-Update request (before approval)
-Cancel request
-Filter by:
-Blood group
-Urgency
-Status
Status Flow:
Pending → Accepted → Completed → Cancelled

5. Blood Donation Module
Features:
-View matching requests (based on blood group)
-Accept donation request
-Confirm donation
-Update donation status
-Donation history tracking

6. Communication Module (Request Comments)
Features:
-Add comment on blood request
-View conversation history

7. Report Module
Features:
-Total donors
-Total requests
-Completed donations
-Pending requests
-Blood group statistics
-Monthly donation report

8. Admin Dashboard Module
Features:
-Dashboard summary cards
-User management shortcut
-Request monitoring
-Donation monitoring
-System health overview

9. Matching Engine (Business Logic Module ⭐)
Rules:
-Match based on BloodGroup
-Check Donor availability
-Check LastDonationDate (optional rule)
-Avoid already donated donors
Example logic:
(Request: A+
Donor: A+ or O+ → eligible)

*System Flow Summary
User Login
   ↓
Role Check
   ↓
Requester → Create Blood Request
   ↓
System Matches Donors
   ↓

   ↓
Donation Completed
   ↓
Report Updated
